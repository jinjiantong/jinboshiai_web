## 一、项目概述

### 1.1 项目背景

金博士AI官网智能客服系统，为用户提供7x24小时的课程咨询和技术答疑服务。基于RAG（检索增强生成）技术，结合知识库向量检索和大语言模型对话能力。

### 1.2 核心功能

| 功能 | 说明 | 状态 |
|------|------|------|
| 智能对话 | RAG检索 + 大模型生成 | ✅ 已上线 |
| 知识库管理 | 课程知识同步到向量库 | ✅ 已上线 |
| 数据统计 | 高频问题统计 | ✅ 已上线 |
| 缓存优化 | Redis缓存加速响应 | ✅ 已上线 |
| 数据持久化 | MySQL存储对话记录 | ✅ 已上线 |

### 1.3 技术栈

| 组件 | 技术选型 | 用途 |
|------|---------|------|
| 前端框架 | Next.js + React | 官网现有技术栈 |
| 样式框架 | Tailwind CSS | 官网现有技术栈 |
| 动画效果 | Framer Motion | 官网现有技术栈 |
| 大模型 | MiniMax API | 对话生成、Embedding |
| 向量数据库 | Qdrant | 语义检索 |
| 业务数据库 | MySQL | 对话历史、知识库备份 |
| 缓存服务 | Redis | 响应缓存、加速 |
| 部署平台 | 腾讯云轻量服务器 | Ubuntu + Docker |

## 四、数据库设计

### 4.1 MySQL表结构

```sql
-- 会话表
CREATE TABLE chat_sessions (
  session_id VARCHAR(64) PRIMARY KEY,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 消息表
CREATE TABLE chat_messages (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  session_id VARCHAR(64) NOT NULL,
  role ENUM('user', 'assistant') NOT NULL,
  content TEXT NOT NULL,
  source_used JSON,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (session_id) REFERENCES chat_sessions(session_id)
);
```

### 4.2 Redis缓存结构

| Key格式 | Value | 过期时间 |
|---------|-------|---------|
| chat:{base64编码问题} | JSON(回答+sources) | 3600秒 |

### 4.3 Qdrant Collection

| chatbot_knowledge | 1536 | Cosine | 10条 |
