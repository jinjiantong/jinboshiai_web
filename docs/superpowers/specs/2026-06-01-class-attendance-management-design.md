## 🎨 功能详细设计

### 1. 顶部提醒区域

**功能**：
- 显示即将上课的班级（当前时间后 30 分钟内）
- 红色高亮醒目提示
- 显示倒计时："距离开课还有 25 分钟"
- 点击可直接跳转到该班级的考勤页面

**逻辑**：
```
获取当前时间
查询所有"上课中"班级
筛选：上课时间段在当前时间 + 30分钟内
按上课时间升序排列
显示前 3 个即将上课的班级
```

### 2. 搜索筛选区域

**筛选条件**：
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| 班级分类 | 单选下拉 | 周末班、周二班、周三班... |
| 日期 | 日期选择器 | 默认当天 |
| 班级状态 | 单选下拉 | 招生中、上课中、已结课 |
| 关键字搜索 | 文本输入 | 按班级名称搜索 |

**联动逻辑**：
- 选择日期后，自动筛选该日期有排课的班级
- 支持组合筛选

### 3. 班级列表区域

**显示内容**：
- 班级名称
- 上课时间段（如 "09:00-12:00"）
- 学生人数
- 状态标签（即将上课 / 正常 / 已结束）

**交互**：
- 点击班级 → 右侧加载该班级的学生列表
- 即将上课的班级高亮显示
- 已考勤过的班级显示"已完成"标记

### 4. 学生考勤区域

**显示内容**：
- 班级信息标题
- 学生卡片列表：
  - 复选框（支持多选）
  - 学生姓名
  - 联系电话
  - 学习状态标签

**交互**：
- 点击学生卡片 → 切换选中状态
- 点击卡片右侧 → 展开更多操作（单独标记迟到/请假/旷课）
- 支持批量操作

### 5. 批量考勤操作

**操作流程**：
```
1. 用户在学生列表中勾选要考勤的学生（支持全选）
2. 点击"批量已上课"按钮
3. 系统执行：
   a. 创建考勤记录（签到状态 = "正常出勤"）
   b. 扣减课时（已上课时 +1）
4. 显示成功提示，刷新页面
```

**确认弹窗**：
```
┌─────────────────────────────────────────┐
│  确认批量考勤？                           │
│                                         │
│  将标记以下 4 位学生为"已上课"：           │
│  • 张三                                  │
│  • 李四                                  │
│  • 王五                                  │
│  • 赵六                                  │
│                                         │
│  每人将扣减 1 课时                       │
│                                         │
│       [取消]        [确认考勤]            │
└─────────────────────────────────────────┘
```

### 6. 单个学生考勤操作

**右键/点击操作**：
| 操作 | 说明 |
| :--- | :--- |
| 已上课 | 正常出勤，扣减 1 课时 |
| 迟到 | 迟到标记，不扣减课时 |
| 请假 | 请假标记，不扣减课时 |
| 旷课 | 旷课标记，扣减 1 课时 |

### API 端点设计

#### 1. 获取班级列表
```
GET /api/class-management/classes
Query 参数：
  - date: 日期 (YYYY-MM-DD)
  - category: 班级分类
  - status: 班级状态
  - keyword: 搜索关键字

返回：
{
  "classes": [
    {
      "record_id": "xxx",
      "class_name": "周末AI班",
      "time_slot": "09:00-12:00",
      "student_count": 12,
      "status": "upcoming" | "ongoing" | "finished"
    }
  ]
}
```

#### 2. 获取班级学生列表
```
GET /api/class-management/classes/{class_id}/students

返回：
{
  "class_info": {
    "name": "周末AI班",
    "time_slot": "09:00-12:00",
    "course": "AI提示词"
  },
  "students": [
    {
      "record_id": "xxx",
      "name": "张三",
      "phone": "138****8000",
      "status": "正常上课",
      "remaining_hours": 15
    }
  ]
}
```

#### 3. 批量考勤操作
```
POST /api/class-management/attendance/batch
Body：
{
  "class_id": "xxx",
  "student_ids": ["xxx", "xxx", "xxx"],
  "action": "present" | "late" | "absent" | "leave",
  "date": "2026-06-01"
}

返回：
{
  "success": true,
  "processed_count": 4,
  "message": "成功标记 4 位学生考勤状态"
}
```

#### 4. 获取即将上课的班级
```
GET /api/class-management/classes/upcoming
Query 参数：
  - minutes: 提前提醒分钟数（默认30）

返回：
{
  "upcoming_classes": [
    {
      "record_id": "xxx",
      "class_name": "周末AI班",
      "time_slot": "09:00-12:00",
      "minutes_until_start": 25
    }
  ]
}
```

## ⚠️ 异常处理

### 1. 课时不足
```
场景：学生剩余课时为 0 或 1
处理：
  - 考勤前提示："学生剩余课时不足，是否继续考勤？"
  - 考勤后记录需要跟进续费
```

### 2. 已考勤检查
```
场景：学生当天已考勤
处理：
  - 显示"已考勤"标签
  - 禁止重复考勤操作
```

### 3. 网络异常
```
场景：API 调用失败
处理：
  - 显示错误提示
  - 提供重试按钮
  - 记录失败操作日志
```

### 4. 班级无排课
```
场景：当天没有班级上课
处理：
  - 显示空状态："今天暂无课程安排"
  - 显示最近一周有课的日期
```

## 🚀 性能优化

### 1. 数据缓存
- 班级列表数据缓存 5 分钟
- 学生列表数据实时获取
- 即将上课提醒每分钟刷新

### 2. 批量操作优化
- 考勤操作合并为单个 API 调用
- 课时扣减使用批量更新

### 3. 加载优化
- 班级列表懒加载
- 学生列表分页（每页 20 人）
