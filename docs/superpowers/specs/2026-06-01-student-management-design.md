## 一、系统概述

学生管理系统是一个独立的Web应用，用于培训机构管理学员档案、班级课程、考勤和缴费等核心业务数据。系统通过飞书多维表格API与后端数据存储同步。

### 1.1 核心功能范围

1. **学员档案管理** - 增删改查学员基本信息
2. **班级课程管理** - 增删改查班级和课程信息
3. **考勤管理** - 记录和查看学员出勤情况
4. **缴费管理** - 管理学员缴费记录

### 1.2 用户角色

- **老师**：拥有所有功能的完整操作权限（增删改查）

### 1.3 技术栈

- 前端框架：Next.js 14 (App Router)
- 样式方案：Tailwind CSS
- 图标库：Lucide React
- 数据交互：飞书多维表格API

## 四、功能模块详细设计

### 4.1 学员档案管理

#### 4.1.1 数据字段

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| 学员ID | auto_number | 自动递增的学员唯一标识 |
| 姓名 | text | 学员姓名 |
| 性别 | select | 男/女 |
| 年龄 | number | 学员年龄 |
| 联系电话 | text | 学员或家长联系电话 |
| 微信 | text | 微信号 |
| 报名日期 | datetime | 报名日期 |
| 来源渠道 | select | 转介绍/线上广告/线下活动/其他 |
| 学习状态 | select | 未开课/正常上课/请假/休学/结业/退学 |
| 学员标签 | select | 零基础/有基础/在职/全职/试听学员 |

#### 4.1.2 功能操作

- **查看列表**：表格视图/卡片视图切换显示
- **搜索**：按姓名、手机号搜索
- **筛选**：按学习状态、学员标签筛选
- **添加**：点击添加按钮，弹出表单弹窗
- **编辑**：点击编辑按钮，弹出预填充表单弹窗
- **删除**：点击删除按钮，确认后删除
- **查看详情**：点击查看按钮，显示学员详情及关联信息

#### 4.1.3 界面布局

**表格视图列**：
- 学员ID
- 姓名
- 性别
- 年龄
- 联系电话
- 学习状态
- 操作（查看/编辑/删除）

**卡片视图布局**：
- 卡片尺寸：3列网格布局
- 卡片内容：姓名、状态标签、联系电话、学习状态
- 卡片底部：操作按钮组

## 五、交互设计

### 5.1 视图切换

- 位于内容区右上角
- 按钮图标：Grid（表格）/LayoutGrid（卡片）
- 切换后保持当前筛选和搜索条件

### 5.2 添加/编辑弹窗

- 居中显示
- 宽度：500px
- 包含表单字段和操作按钮
- 点击遮罩或取消按钮关闭
- 提交后自动刷新列表

### 5.3 删除确认

- 使用确认对话框
- 显示删除对象的名称
- 确认后执行删除操作

### 5.4 加载状态

- 数据加载中显示骨架屏
- 操作提交中按钮显示loading状态

### 5.5 错误处理

- API错误显示错误提示
- 表单验证错误显示字段下方

## 六、API设计

### 6.1 学员档案API

```
GET    /api/students          - 获取学员列表
GET    /api/students/:id      - 获取单个学员详情
POST   /api/students          - 创建新学员
PUT    /api/students/:id      - 更新学员信息
DELETE /api/students/:id      - 删除学员
```

### 6.2 课程班级API

```
GET    /api/courses           - 获取课程列表
POST   /api/courses           - 创建新课程
PUT    /api/courses/:id       - 更新课程信息
DELETE /api/courses/:id       - 删除课程
GET    /api/classes           - 获取班级列表
POST   /api/classes           - 创建新班级
PUT    /api/classes/:id       - 更新班级信息
DELETE /api/classes/:id       - 删除班级
```

### 6.3 考勤管理API

```
GET    /api/attendance        - 获取考勤记录
POST   /api/attendance        - 创建考勤记录
PUT    /api/attendance/:id    - 更新考勤状态
DELETE /api/attendance/:id    - 删除考勤记录
```

### 6.4 缴费管理API

```
GET    /api/payments          - 获取缴费记录
POST   /api/payments          - 创建缴费记录
PUT    /api/payments/:id      - 更新缴费信息
DELETE /api/payments/:id      - 删除缴费记录
```

## 八、技术注意事项

1. **数据同步**：所有操作通过飞书多维表格API实时同步
2. **权限控制**：仅老师角色可进行增删改操作
3. **表单验证**：必填字段必须填写，格式校验
4. **用户反馈**：操作成功后显示成功提示，失败显示错误信息
5. **性能优化**：大列表数据使用分页加载
