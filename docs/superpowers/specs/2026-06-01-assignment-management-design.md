## 1. 项目概述

### 1.1 项目目标

开发一个作业管理系统，集成到现有的学生管理后台中，支持学员提交作业和老师管理作品。

### 1.2 核心功能

- 作品展示：师生都能查看所有学生的作品
- 作品管理：学生只能操作自己的作品，老师可以操作所有作品
- 作品提交：支持随时提交，无需截止日期限制
- 状态管理：仅支持提交状态（已提交/未提交），无需评分功能

### 1.3 权限模型

| 角色 | 查看权限 | 操作权限 |
| :--- | :--- | :--- |
| 学生 | 所有作品 | 仅自己的作品 |
| 老师 | 所有作品 | 所有作品 |

## 5. 用户流程

### 5.1 页面加载流程

1. 检查用户登录状态（从 localStorage 或 session）
2. 如果未登录，重定向到登录页面
3. 获取用户信息（type、id、name）
4. 根据用户类型显示不同界面
5. 调用 API 获取作业列表
6. 渲染作品展示区域

### 5.2 新增作品流程

```
点击"新增作品"按钮
    ↓
弹出 Modal 表单
    ↓
填写作品信息
    ↓
上传附件（可选）
    ↓
点击"提交"按钮
    ↓
前端验证
    ↓
调用 POST /api/assignments
    ↓
保存到多维表格
    ↓
关闭 Modal，刷新列表
    ↓
显示成功提示
```

### 5.3 编辑作品流程

```
点击作品卡片上的"编辑"按钮
    ↓
弹出 Modal 表单（预填数据）
    ↓
修改作品信息
    ↓
点击"保存"按钮
    ↓
前端验证
    ↓
调用 PUT /api/assignments/:id
    ↓
更新多维表格记录
    ↓
关闭 Modal，刷新列表
    ↓
显示成功提示
```

### 5.4 删除作品流程

```
点击作品卡片上的"删除"按钮
    ↓
弹出确认对话框
    ↓
点击"确认删除"按钮
    ↓
调用 DELETE /api/assignments/:id
    ↓
从多维表格删除记录
    ↓
刷新列表
    ↓
显示成功提示
```

## 6. 权限控制

### 6.1 权限判断逻辑

```typescript
// 判断用户是否已登录
const isLoggedIn = () => {
  const user = getUserFromStorage();
  return user !== null;
};

// 判断用户是否为老师
const isTeacher = () => {
  const user = getUserFromStorage();
  return user?.type === 'teacher';
};

// 判断用户是否能操作某作品
const canOperate = (assignment: Assignment, user: User) => {
  if (user.type === 'teacher') {
    return true;  // 老师可以操作所有作品
  }
  return assignment.关联学员 === user.id;  // 学生只能操作自己的作品
};

// 判断用户是否能标记优秀
const canMarkExcellent = (user: User) => {
  return user.type === 'teacher';
};
```

### 6.2 按钮显示控制

```typescript
// 根据权限显示操作按钮
const ActionButtons = ({ assignment, user }) => {
  const canEdit = canOperate(assignment, user);
  const canDelete = canOperate(assignment, user);
  const canMark = canMarkExcellent(user);

  return (
    <>
      {canEdit && <Button>编辑</Button>}
      {canDelete && <Button>删除</Button>}
      {canMark && <Button>标记优秀</Button>}
    </>
  );
};
```

## 7. 错误处理

### 7.1 网络错误

```typescript
try {
  const response = await fetch('/api/assignments');
  if (!response.ok) {
    throw new Error('网络请求失败');
  }
} catch (error) {
  showToast('网络错误，请稍后重试', 'error');
}
```

### 7.2 加载失败

- 显示错误提示和重试按钮
- 自动重试机制（最多 3 次）

### 7.3 操作失败

- 显示具体错误信息
- 保留表单数据，方便用户重新提交

### 7.4 Toast 提示

使用现有的 Toast 组件（`/app/components/ui/Toast.tsx`）显示操作结果。
