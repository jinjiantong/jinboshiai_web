## 1. 项目概述

### 1.1 项目简介

金博士AI官网是一个面向零基础用户的AI实战教育培训平台官网，采用现代化前端技术栈构建，为访客提供课程展示、师资介绍、在线报名等功能，同时为注册学员和教师提供后台管理系统入口。

### 1.2 项目定位

- **面向用户**：零基础AI学习者、有进阶需求的职场人士、对AI感兴趣的企业用户
- **核心价值**：展示AI培训课程、传递品牌理念、提供学员登录入口
- **业务目标**：获取潜在学员咨询线索、建立品牌形象、为后台系统导流

### 1.3 技术选型原则

| 原则 | 说明 |
|------|------|
| 现代化 | 采用业界主流技术栈，确保技术先进性和可维护性 |
| 高性能 | 利用服务端渲染和静态生成，提升首屏加载速度 |
| 易扩展 | 模块化架构设计，便于功能迭代和系统扩展 |
| 低成本 | 依托飞书多维表格作为数据后端，降低运维成本 |

## 5. API接口设计

### 5.1 API路由总览

| 模块 | 路由 | 方法 | 说明 |
|------|------|------|------|
| **作品展示** | `/api/portfolios` | GET | 获取作品列表 |
| **登录验证** | `/api/login` | POST | 用户登录 |
| **班级查询** | `/api/classes` | GET | 获取班级列表 |
| **数据统计** | `/api/stats` | GET | 获取统计数据 |
| **学员管理** | `/api/student-management/students` | GET/POST/PUT/DELETE | 学员CRUD |
| **教师管理** | `/api/student-management/teachers` | GET/POST/PUT/DELETE | 教师CRUD |
| **班级管理** | `/api/student-management/courses` | GET/POST/PUT/DELETE | 班级CRUD |
| **考勤管理** | `/api/student-management/attendance` | GET/POST/PUT/DELETE | 考勤CRUD |
| **作业管理** | `/api/assignments` | GET/POST | 作业列表/创建 |

### 5.2 认证机制

```typescript
// 登录流程
POST /api/login
Request: { type: 'teacher' | 'student', username: string, classId?: string }
Response: { success: boolean, message: string, data?: { type, name, id, classId? } }
```

**权限控制**：
- 教师：可访问所有管理功能
- 学员：仅可访问自己相关数据
- 未登录：重定向至首页

## 8. 性能优化

### 8.1 首屏加载优化

| 优化项 | 实现方式 | 预期效果 |
|--------|----------|----------|
| SSR | Next.js服务端渲染 | 快速显示内容 |
| 图片优化 | WebP格式 + 懒加载 | 减少带宽占用 |
| 代码分割 | 自动路由级分割 | 减少初始加载体积 |
| CDN | 静态资源托管 | 加速全球访问 |

### 8.2 运行性能优化

| 优化项 | 实现方式 |
|--------|----------|
| 缓存策略 | API响应缓存、Token缓存 |
| 防抖节流 | 搜索输入、滚动事件 |
| 虚拟列表 | 长列表渲染优化 |
| 组件懒加载 | 非首屏组件延迟加载 |
